Financial Invariants

Four Non-Negotiable Properties

Sum Dispersal = Distributable

Sum of all rounded per-lender dispersal outputs must exactly equal the total distributable amount. No cents lost or created.

T-019

1 Confirmation = 1 Posting

A single FUNDS_SETTLED event produces exactly one cash ledger journal entry. Not zero, not two.

T-020

Replay = Zero Effect

Processing the same webhook event twice must not create additional ledger entries. State machine rejects; idempotency key deduplicates.

T-021

Reversal Net = Zero

After reversal, the net effect on all accounts is zero. The compensating entry exactly cancels the original.

T-022

Notes

Implementation Guidance

Backward Compatibility

All new tests are additions only. No existing test file may be modified. The D4 conditional ensures bridge transfers coexist with legacy collection attempt cash posting.

Critical: Hash Chain Disabled in Tests

All integration tests must set process.env.DISABLE_GT_HASHCHAIN = "true" before creating the convex-test harness.

Test Harness Pattern

Use convexTest(schema, modules) with auditLogTest.register(t, "auditLog"). Seed entities via t.run(). Identity injection via t.withIdentity().

Quality Gates

After each chunk: bun check + bun typecheck + bunx convex codegen. After chunk 5: full bun run test must be green.
