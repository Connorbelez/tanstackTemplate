# Admin origination parallelized spec package

Files in this directory:

- `mockups/` — production-style **HTML/CSS** screen reference for all phases (`index.html`, `phase-01.html` … `phase-09.html`). See `mockups/README.md` for how to open locally and how to mirror the same screens in **Paper** via MCP when Paper Desktop is connected.
- `admin-origination-parallelized-spec-package.md` — combined package containing Deliverables 1, 2, 3, and 4 in order.
- `phase-01-origination-case-scaffold-ui-skeleton.md`
- `phase-02-canonical-borrower-property-mortgage-activation-without-payments-or-docs.md`
- `phase-03-listing-projection-and-public-doc-compatibility-projection.md`
- `phase-04-payment-bootstrap-integration.md`
- `phase-05-immediate-rotessa-activation.md`
- `phase-06-mortgage-document-blueprints-and-public-private-static-docs.md`
- `phase-07-deal-time-package-materialization-for-private-static-and-non-signable-templated-docs.md`
- `phase-08-signable-docs-documenso-envelopes-and-embedded-signing.md`
- `phase-09-signed-archive-broker-deal-visibility-and-demo-hardening.md`

The phase files are intended for isolated handoff to parallel coding agents. The combined package is intended for program lead / merge-manager review.

This package intentionally preserves the nine-phase structure while correcting production-safety gaps found during the codebase audit. Treat the combined package and the standalone phase files as synchronized artifacts: if ownership, invariants, or contracts change in one, they must be updated in the other in the same edit.
